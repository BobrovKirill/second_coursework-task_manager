# API Documentation

## Users & Auth

**Дата:** 19.05.2026

---

### Пользователи (`/users`)

| Эндпоинт           | Метод  | CRUD | Автор   | Описание                                     | Примечания                  |
|--------------------|--------|------|---------|----------------------------------------------|-----------------------------|
| `/users/`          | POST   | Да   | Кирилл  | Создание нового пользователя                 | -                           |
| `/users/{user_id}` | GET    | Да   | Кирилл  | Получение пользователя по ID                 | -                           |
| `/users/`          | GET    | Да   | Кирилл  | Получение списка пользователей               | `skip`, `limit` (пагинация) |
| `/users/{user_id}` | DELETE | Да   | Кирилл  | Удаление пользователя                        | -                           |
| `/users/me`        | GET    | -    | Кирилл  | Получение профиля текущего пользователя      | Поддерживает `?project_id=` |
| `/users/me`        | PATCH  | -    | Кирилл  | Обновление профиля текущего пользователя     | Поддерживает `?project_id=` |
| `/users/avatar`    | POST   | -    | Кирилл  | загрузка аватара в хранилище, возвращает url | -                           |
| `/users/search`    | GET    | -    | Кирилл  | Поиск пользователей                          | `email`, `full_name`, `project_id` |

**Особенности эндпоинтов `/users/me`:**

- При передаче query-параметра `project_id` дополнительно возвращается:
  - Поле `role` — роль пользователя в проекте
  - Поле `permissions` — права пользователя в данном проекте

### Авторизация (`/auth`)

| Эндпоинт                       | Метод | CRUD | Автор   | Описание                           | Примечания                         |
|--------------------------------|-------|------|---------|------------------------------------|------------------------------------|
| `/auth/login`                  | POST  | -    | Кирилл  | Авторизация пользователя           | Возвращает JWT access token        |
| `/auth/verify-email`           | POST  | -    | Кирилл  | Подтверждение email при регистрации | Активирует пользователя по токену  |
| `/auth/password-reset/request` | POST  | -    | Кирилл  | Запрос письма восстановления пароля | Всегда возвращает одинаковый ответ |
| `/auth/password-reset/confirm` | POST  | -    | Кирилл  | Установка нового пароля             | Проверяет одноразовый токен        |
| `/auth/project-invitations/accept` | POST | - | Кирилл | Принятие приглашения в проект | Создает пользователя и добавляет в проект |

**Тело запроса (`LoginRequest`):**
- `email`
- `password`

**Тело запроса (`EmailVerificationRequest`):**
- `token`

**Тело запроса (`PasswordResetRequest`):**
- `email`

**Тело запроса (`PasswordResetConfirm`):**
- `token`
- `password`
- `confirm`

**Тело запроса (`ProjectInvitationAccept`):**
- `token`
- `password`
- `confirm`

**Возможные ошибки:**
- `401 Unauthorized` — Неверный email или пароль
- `400 Bad Request` — Пользователь неактивен (`is_active = false`)
- `400 Bad Request` — Ссылка подтверждения email недействительна или устарела
- `400 Bad Request` — Ссылка восстановления пароля недействительна или устарела
- `400 Bad Request` — Ссылка приглашения недействительна или устарела

### Задачи (`/projects/{project_id}/tasks`, `/tasks`)

| Эндпоинт                        | Метод  | CRUD | Автор  | Описание                        | Примечания                                                      |
|---------------------------------|--------|------|--------|---------------------------------|-----------------------------------------------------------------|
| `/projects/{project_id}/tasks`  | POST   | Да   | Ксения | Создание задачи в проекте       | Сейчас доступно владельцу проекта                               |
| `/projects/{project_id}/tasks`  | GET    | Да   | Ксения | Получение списка задач проекта  | Доступно владельцу и участникам проекта                         |
| `/tasks/me`                     | GET    | -    | Кирилл | Получение задач текущего пользователя | Возвращает назначенные на пользователя задачи со сведениями о проекте |
| `/tasks/{task_id}`              | GET    | Да   | Ксения | Получение одной задачи          | Доступно владельцу и участникам проекта                         | 
| `/tasks/{task_id}`              | PUT    | Да   | Ксения | Редактирование задачи           | Доступно владельцу проекта, admin, organizer и создателю задачи |
| `/tasks/{task_id}`              | DELETE | Да   | Ксения | Удаление задачи                 | Доступно владельцу проекта, admin, organizer и создателю задачи |

**Ответ `/tasks/me`:**
- `id`
- `title`
- `projectId`
- `projectName`
- `status`
- `priority`
- `deadline`

**Возможные ошибки:**
- `403 Forbidden` — нет доступа к проекту или задаче
- `404 Not Found` — проект или задача не найдены

### Приглашения в проект (`/projects/{project_id}/invitations`)

| Эндпоинт                          | Метод | CRUD | Автор   | Описание                         | Примечания |
|-----------------------------------|-------|------|---------|----------------------------------|------------|
| `/projects/{project_id}/invitations` | POST | - | Кирилл | Приглашение участника по email | Если пользователь уже существует, добавляется без письма |

**Тело запроса (`ProjectInvitationCreate`):**
- `email`
- `role`
- `specialty`

**Особенности:**
- Если пользователь с таким email уже есть в базе, backend сразу добавляет его в проект с указанной ролью и специализацией.
- Если пользователя нет в базе, backend отправляет письмо со ссылкой принятия приглашения.
- После принятия приглашения пользователь задает пароль, создается активный аккаунт, и пользователь добавляется в проект.
