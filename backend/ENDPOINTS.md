# API Documentation

## Users & Auth

**Последнее обновление:** 19.05.2026

---

### Пользователи (`/users`)

| Метод | Эндпоинт           | CRUD | Описание                              | Примечания |
|-------|--------------------|------|---------------------------------------|----------|
| POST  | `/users/`          | Да   | Создание нового пользователя          | - |
| GET   | `/users/{user_id}` | Да   | Получение пользователя по ID          | - |
| GET   | `/users/`          | Да   | Получение списка пользователей        | `skip`, `limit` (пагинация) |
| DELETE| `/users/{user_id}` | Да   | Удаление пользователя                 | - |
| GET   | `/users/me`        | -    | Получение профиля текущего пользователя | Поддерживает query-параметр `project_id` |
| PATCH | `/users/me`        | -    | Обновление профиля текущего пользователя | Поддерживает query-параметр `project_id` |

**Особенности `/users/me`:**

- При передаче `project_id` в query-параметре дополнительно возвращается:
  - Роль пользователя в указанном проекте (`role`)
  - Права пользователя в проекте (`permissions`)

### Авторизация (`/auth`)

| Метод | Эндпоинт       | Описание                  | Примечания |
|-------|----------------|---------------------------|----------|
| POST  | `/auth/login`  | Авторизация по email + пароль | Возвращает `access_token` (JWT) |

**Тело запроса (`LoginRequest`):**
- `email`
- `password`

**Возможные ошибки:**
- `401` — Неверный email или пароль
- `400` — Пользователь неактивен (`is_active = false`)
